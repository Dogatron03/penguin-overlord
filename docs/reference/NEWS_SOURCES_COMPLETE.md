- Failed feeds don't crash the bot
- Timeouts after 10 seconds
- Automatic retry on next scheduled run
- Comprehensive logging

### âœ… Deduplication
- Each category maintains a state file
- Tracks posted items by GUID/URL
- Never posts the same item twice

### âœ… Date Filtering
- Only posts items from last 7 days
- Prevents old news from flooding channels
- Configurable per category

### âœ… ETag Caching
- Stores HTTP ETag headers
- Sends conditional requests
- Server returns 304 Not Modified if unchanged
- Reduces bandwidth and improves performance

### âœ… Rate Limiting

- Prevents overwhelming Discord API
- Configurable concurrency limits

### âœ… HTML Processing

- Unescapes HTML entities (&nbsp;, &lt;, etc.)

## Feed Health Monitoring

### Check Feed Status

```bash
# Test a single feed
curl -s -o /dev/null -w "HTTP %{http_code}\n" "https://krebsonsecurity.com/feed/"

# Test multiple feeds
for url in $(cat feed_urls.txt); do
    echo "Testing: $url"
    curl -s -o /dev/null -w "HTTP %{http_code}\n" "$url"
done
```

### View Logs

```bash
# View recent activity
journalctl -u penguin-news-cybersecurity -n 50

# Watch for errors
journalctl -u penguin-news-* -f | grep -E "ERROR|WARNING"

# Check all news services
systemctl status 'penguin-news-*'
```
